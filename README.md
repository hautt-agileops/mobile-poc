# mobile-poc

Proof-of-concept workspace for AI-driven game and 3D asset pipelines. Each POC is independent — no shared build system or deps.

## Skills

All Claude Code skills live in `.claude/skills/` and are invoked via `/skill-name` in Claude Code.

| skill | what it does |
|-------|-------------|
| `3d-prompt` | Job brief → 4-view PNGs → GLB/STL (Meshy) → optional public Vercel gallery |
| `unity-poc` | Game brief → design docs → code-driven Unity WebGL build → public Vercel URL |
| `game-asset-gen` | Asset manifest → 2D sprites (Vertex AI) + 3D models (Meshy GLB) |
| `game-design-document` | Author PRD / GDD from a brief |

## Outputs

- `unity/output/<project>/` — per-game design docs, build logs, and `HANDOFF.md` (gitignored; regenerated by pipeline)

## Requirements

- Node 18+
- Unity 6000.x + WebGL module (for `unity-poc`)
- Vercel CLI (`npm i -g vercel`)
- Vertex AI service account (for image gen; falls back to programmer art if missing)
- Meshy API key (for 3D models)

See `CLAUDE.md` for full architecture, credential resolution, and pipeline gotchas.
