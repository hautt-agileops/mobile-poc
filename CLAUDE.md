## 3d-prompt skill

Worker-less pipeline: brief → refined visual prompt → 4 view PNGs → 3D model (GLB/STL/…), optionally published to Vercel gallery. Pure Node 18+ built-ins — no npm deps, no server.

### Architecture (`.claude/skills/3d-prompt/`)

- **`pipeline.mjs`** (~500 LOC). Flow:
  1. `refinePrompt()` — Vertex AI Gemini text rewrites brief into a visual prompt (skip with `-n`).
  2. `generateViews()` — multi-turn Gemini image gen for 4 consistent views (`front, left side, back, right side`); prior views fed back as context. `resolvePose()` picks A-pose for humanoid subjects via `HUMANOID_KEYWORDS`.
  3. `createMeshyTask()` → `waitForMeshy()` — submits 4 PNGs to Meshy `multi-image-to-3d` (`meshy-6`), polls, downloads formats.
  4. Auto-invokes `publish.mjs` unless `-P`.
  - Auth: `getAccessToken()` mints a `cloud-platform` OAuth token by hand-signing an RS256 JWT via Node `crypto` — no google-auth lib.
- **`publish.mjs`** (~150 LOC) — bundles `model.*` + 4 PNGs into `viewer/models/<task-id>/`, rebuilds `viewer/manifest.json`, deploys `viewer/` to Vercel.
- **`viewer/`** — static `<model-viewer>` gallery (`index.html` + `vercel.json`). Fully public.
- **`SKILL.md`** — skill contract.

### Models / providers

- Text: `gemini-3.1-pro-preview` (override: `GEMINI_TEXT_MODEL`)
- Image: `gemini-3-pro-image-preview` → mapped to `gemini-3.1-flash-image` on Vertex (override: `GEMINI_IMAGE_MODEL`)
- 3D: Meshy only (`multi-image-to-3d`, `meshy-6`). Vertex AI only — no Gemini Developer API-key path.

### Run

```bash
cd .claude/skills/3d-prompt
node pipeline.mjs "a cute robot"                          # quick smoke test
node pipeline.mjs -n -f stl,glb -J <job-file> -m "<notes>" "<prompt>"
node pipeline.mjs -h                                      # all flags
node publish.mjs <task-id>                                # (re)publish existing output
node publish.mjs <task-id> -n                             # bundle + manifest only, no deploy
cd viewer && python3 -m http.server 8000                  # local viewer preview
```

Flags: `-n` no-refine, `-i` reference image (local path or https URL), `-f` formats (`glb` always added), `-m` notes digest, `-J` archive brief, `-o` output base (default `./output`), `-P` skip auto-deploy. Output: `output/<task-id>/` (4 PNGs, `model.glb`, `prompt.txt`, `job.txt`).

### Credentials (auto-resolved — do NOT ask user to export vars first)

- **Vertex SA**: `sa.json` next to the skill or in cwd → `GOOGLE_APPLICATION_CREDENTIALS` → `GOOGLE_SA_OP_REF` (1Password) → `GOOGLE_SERVICE_ACCOUNT_JSON` (inline single-quoted) → `GOOGLE_VERTEX_ACCESS_TOKEN`. Project auto-derives from SA `project_id`.
- **Meshy**: `MESHY_API_KEY` or pulled from 1Password (`MESHY_OP_REF`).
- Only ask for credentials if BOTH SA and 1Password are unavailable.

### Gotchas

- Never load SA via `set -a; source` — bash strips JSON double quotes. Use `sa.json` file or single-quoted inline value.
- First Vercel deploy requires one-time `cd viewer && npx vercel login && npx vercel link`.

## unity-poc pipeline

Worker-less: game brief → design docs → code-driven Unity project → WebGL build → public Vercel URL. Everything spawns from C# at runtime (one empty boot scene, no hand-authored scenes or prefabs). Ships two bundled fighter frameworks.

### Pipeline phases (load sub-skills in order)

| phase | sub-skill | steps | output |
|-------|-----------|-------|--------|
| Spec | `unity-poc-spec` | 1–3 | `PRD.md` → `TDD.md` → `ASSETS.md` / `assets.manifest.json` / `models.manifest.json` |
| Scaffold | `unity-poc-scaffold` | 4–5 | Unity project created headlessly; `template/` or `template3d/` copied |
| Assets | `unity-poc-assets` | 6 | PNGs → `Resources/Art/` (2D); GLBs → `Resources/Models/` (3D). Never a hard gate — missing art falls back to flat color / primitive capsule |
| Gameplay | `unity-poc-gameplay` | 7 | `Assets/Scripts/Game/` authored; `BuildRoster()` exposed |
| Build+Ship | `unity-poc-buildship` | 8–13 | Headless playtest gate → WebGL build → local puppeteer boot test → Vercel deploy → `HANDOFF.md` |

### Template pick

| brief | template | namespace | assets |
|-------|----------|-----------|--------|
| 2D fighter / arcade | `template/` | `Fighter` | 2D PNG sprites (`game-asset-gen`) |
| 3D arena brawler | `template3d/` | `Fighter3D` | Meshy GLB models (`gen-models.mjs` → glTFast) |
| platformer / cozy / other | — | your own | write from scratch; reuse pipeline + scripts only |

### Key gotchas

- **IL2CPP stripping** — "Could not produce class with ID N" at boot. Build uses `ManagedStrippingLevel.Minimal` + `stripEngineCode = false`. Only the puppeteer browser test catches this.
- **nano-banana never returns real alpha** — run `alpha_key.py` (REQUIRED for transparent sprites); skipping shows grey checkerboard boxes.
- **WebGL compression Disabled** — intentional for plain static hosting.
- **Generated art must live under `Resources/Art/`** (2D) or `Resources/Models/<id>.bytes` (3D) — only `Resources/` ships with `Resources.Load`.
- **glTFast is optional** — real 3D models need BOTH `com.unity.cloud.gltfast` package AND `Assets/csc.rsp` with `-define:HAS_GLTFAST`. Add the define only when the package is present.
- **`*.sh` scripts default to `Fighter.*`** — 3D builds must set `BUILD_METHOD`/`PLAYTEST_METHOD=Fighter3D.*`.
- Two hard gates: **playtest** (aborts build on fail) and **local puppeteer browser test** (aborts deploy on fail). Both fail back to `unity-poc-gameplay`.

### Output location

`unity/output/<project-name>/` — gitignored, regenerated by pipeline. Each project folder contains `PRD.md`, `TDD.md`, `ASSETS.md`, `HANDOFF.md`, build logs, and the asset manifest.
