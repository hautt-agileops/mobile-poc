# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository shape

Root is a container for independent proof-of-concept projects. They share no build system, deps, or tooling. Skills are installed at the root `.claude/skills/` and apply to all POCs.

Git-tracked at root: `README.md`, `CLAUDE.md`, `skills-lock.json`, `.gitignore`.
Not tracked: `unity/output/` (gitignored ‚Äî regenerated by pipeline).

`skills-lock.json` is currently an empty stub (`{"version":1,"skills":{}}`) ‚Äî no pinned skill
versions yet. Populate it if/when skill provenance needs locking.

**Two output roots** (both gitignored, don't conflate):
- `.claude/skills/3d-prompt/output/<task-id>/` ‚Äî skill-local, standalone 3D runs.
- `unity/output/<project-name>/` ‚Äî root-level, full unity-poc projects.

## Skills vs agents (architecture)

Two execution surfaces. **Skills** (`.claude/skills/`) load into the **main loop** ‚Äî use for
interactive work (`AskUserQuestion`), code authoring that wants tool feedback, orchestrators,
and pure knowledge/reference. **Agents** (`.claude/agents/`) run in an **isolated context** and
return a conclusion ‚Äî use for non-interactive execution whose noisy logs (builds, deploys,
asset gen) must stay off the main thread, and for one-shot expert consults. Split rule:
*interactive or authoring ‚Üí skill; non-interactive execution ‚Üí agent.* Constraint: **a subagent
cannot spawn another subagent** (no nesting) ‚Äî so agents run shared scripts directly rather than
delegating to sibling agents.

### Installed skills (`.claude/skills/`)

| skill | purpose |
|-------|---------|
| `3d-prompt` | Brief ‚Üí refined prompt ‚Üí 4 view PNGs ‚Üí GLB/STL via Meshy, optionally published to Vercel gallery |
| `unity-poc` | Orchestrator: game brief ‚Üí design docs ‚Üí Unity WebGL ‚Üí public Vercel URL. Spawns the phase agents below |
| `unity-poc-spec` / `unity-poc-gameplay` | Interactive phase skills for `unity-poc` (steps 1‚Äì3 scope+design; step 7 code authoring) |
| `game-asset-gen` | Manifest ‚Üí 2D sprites (Vertex AI) + 3D models (Meshy GLB) |
| `game-design` / `game-design-document` | PRD/GDD authoring (GDD has a mandatory interactive interview ‚Üí stays a skill) |

### Installed agents (`.claude/agents/`)

| agent | purpose |
|-------|---------|
| `portal-deploy` | Publish a 3D model OR Unity WebGL build to the shared Studio portal (Vercel), return the public deep link |
| `unity-advisor` | Unity C#/rendering/perf expert consult (was the `unity-developer` skill) ‚Äî returns guidance + code, not a build |
| `unity-scaffold` | `unity-poc` phase (steps 4‚Äì5): env check + headless project create. Spawned by the orchestrator |
| `unity-assets` | `unity-poc` phase (step 6): gen 2D PNGs / 3D GLBs from the manifest. Spawned by the orchestrator |
| `unity-buildship` | `unity-poc` phase (steps 8‚Äì13): playtest gate ‚Üí WebGL build ‚Üí browser test ‚Üí `deploy-vercel.sh` ‚Üí handoff. Spawned by the orchestrator |

## Studio portal (shared infra)

Both pipelines publish to ONE public Vercel site ‚Äî the **Studio portal**
(`https://3d-viewer-navy.vercel.app`). It is NOT a per-skill gallery; it is shared infra with
two category dirs, each just `<category>/{manifest.json, <id>.png, ‚Ä¶}`:

| category | on-disk dir | producer | content |
|----------|-------------|----------|---------|
| gallery | `models/` | `3d-prompt` skill (`publish.mjs`) | 3D models (GLB/STL + 4 view PNGs) |
| games | `games/` | `unity-poc` (`deploy-vercel.sh`) / `portal-deploy` agent | WebGL builds |

Portal source lives at `.claude/skills/3d-prompt/viewer/` (static `<model-viewer>` + game
launcher, `index.html` + `vercel.json`). Three writers, one site:
- `3d-prompt/publish.mjs` ‚Äî bundles into `models/`, rebuilds `models/manifest.json`.
- `unity-poc/scripts/deploy-vercel.sh` ‚Äî registers under `games/` (`PORTAL_CATEGORY`, default
  `games`); best-effort, never fails the build deploy.
- `portal-deploy` agent ‚Äî standalone publishes from the main loop; classifies artifact into the
  right category (a subagent can't spawn it, so `unity-buildship` calls the `.sh` directly).

First deploy: one-time `cd viewer && npx vercel login && npx vercel link`.

## 3d-prompt skill

Worker-less pipeline: brief ‚Üí refined visual prompt ‚Üí 4 view PNGs ‚Üí 3D model (GLB/STL/‚Ä¶), optionally published to Vercel gallery. Pure Node 18+ built-ins ‚Äî no npm deps, no server.

### Architecture (`.claude/skills/3d-prompt/`)

- **`pipeline.mjs`** (~500 LOC). Flow:
  1. `refinePrompt()` ‚Äî Vertex AI Gemini text rewrites brief into a visual prompt (skip with `-n`).
  2. `generateViews()` ‚Äî multi-turn Gemini image gen for 4 consistent views (`front, left side, back, right side`); prior views fed back as context. `resolvePose()` picks A-pose for humanoid subjects via `HUMANOID_KEYWORDS`.
  3. `createMeshyTask()` ‚Üí `waitForMeshy()` ‚Äî submits 4 PNGs to Meshy `multi-image-to-3d` (`meshy-6`), polls, downloads formats.
  4. Auto-invokes `publish.mjs` unless `-P`.
  - Auth: `getAccessToken()` mints a `cloud-platform` OAuth token by hand-signing an RS256 JWT via Node `crypto` ‚Äî no google-auth lib.
- **`publish.mjs`** (~150 LOC) ‚Äî bundles `model.*` + 4 PNGs into `viewer/models/<task-id>/`, rebuilds `models/manifest.json`, deploys `viewer/` to Vercel.
- **`viewer/`** ‚Äî the shared **Studio portal** (see "Studio portal" above), NOT a 3d-prompt-only gallery. This skill writes the `gallery`/`models/` category; `unity-poc` writes `games/`. Fully public.
- **`SKILL.md`** ‚Äî skill contract.

### Models / providers

- Text: `gemini-3.1-pro-preview` (override: `GEMINI_TEXT_MODEL`)
- Image: `gemini-3-pro-image-preview` ‚Üí mapped to `gemini-3.1-flash-image` on Vertex (override: `GEMINI_IMAGE_MODEL`)
- 3D: Meshy only (`multi-image-to-3d`, `meshy-6`). Vertex AI only ‚Äî no Gemini Developer API-key path.

### Run

```bash
cd .claude/skills/3d-prompt
node pipeline.mjs "a cute robot"                          # quick smoke test
node pipeline.mjs -n -f stl,glb -J <job-file> -m "<notes>" "<prompt>"
node pipeline.mjs -h                                      # all flags
node publish.mjs <task-id>                                # (re)publish existing output
node publish.mjs <task-id> -n                             # bundle + manifest only, no deploy
cd viewer && python3 -m http.server 8000                  # local viewer preview
```

Flags: `-n` no-refine, `-i` reference image (local path or https URL), `-f` formats (`glb` always added), `-m` notes digest, `-J` archive brief, `-o` output base (default `./output`), `-P` skip auto-deploy. Output: `output/<task-id>/` (4 PNGs, `model.glb`, `prompt.txt`, `job.txt`).

### Credentials (auto-resolved ‚Äî do NOT ask user to export vars first)

- **Vertex SA**: `sa.json` next to the skill or in cwd ‚Üí `GOOGLE_APPLICATION_CREDENTIALS` ‚Üí `GOOGLE_SA_OP_REF` (1Password) ‚Üí `GOOGLE_SERVICE_ACCOUNT_JSON` (inline single-quoted) ‚Üí `GOOGLE_VERTEX_ACCESS_TOKEN`. Project auto-derives from SA `project_id`.
- **Meshy**: `MESHY_API_KEY` or pulled from 1Password (`MESHY_OP_REF`).
- Only ask for credentials if BOTH SA and 1Password are unavailable.

### Gotchas

- Never load SA via `set -a; source` ‚Äî bash strips JSON double quotes. Use `sa.json` file or single-quoted inline value.
- First Vercel deploy requires one-time `cd viewer && npx vercel login && npx vercel link`.

## game-asset-gen skill

Shared asset layer for the unity-poc pipeline (and any manifest-driven pipeline). Worker-less,
zero npm deps (Node 18+). **Reuses `3d-prompt`'s credential resolution verbatim** (`sa.json` /
`GOOGLE_*` / 1Password) ‚Äî same Vertex SA. Owns four scripts + a Python alpha pass + a vendored
sprite pipeline:

- **`gen-assets.mjs`** ‚Äî `assets.manifest.json` ‚Üí 2D PNG sprites/tiles/UI/FX via Vertex AI
  nano-banana (`gemini-3.1-flash-image`). Writes straight into `Assets/Resources/Art/`. Detects
  animation intent ‚Üí spritesheets vs single sprites.
  `node gen-assets.mjs <manifest> [-o dir] [-i idsubstr] [-F force] [-d dry]`
- **`gen-sprites.mjs`** ‚Äî `sprites.manifest.json` (`characters[]`) ‚Üí animated **character
  roster**. One Vertex base idle image per character + ONE wide call per animation state,
  sliced into per-frame alpha PNGs (`<id>_<state>_<n>.png` + `<id>.png`) by the vendored
  `sprite-gen` pipeline. ~7 calls/char vs ~17 per-frame, stronger identity lock, **real alpha
  (no `alpha_key.py`)**. Needs python3 + Pillow. Uses SKILL-local **`slot_extract.py`** (forced
  even-slot slicing) NOT sprite-gen's connected-component extractor ‚Äî a fighter's weapon bridges
  slots and fuses poses into one blob. `vendor/sprite-gen/` = aldegad/sprite-gen (Apache-2.0).
  Roster ‚Üí this; stages/UI/FX ‚Üí `gen-assets.mjs`. `node gen-sprites.mjs <manifest> [-i id] [-F] [-d] [--curate]`
- **`gen-models.mjs`** ‚Äî `models.manifest.json` ‚Üí 3D GLBs by **delegating to `3d-prompt/pipeline.mjs`**
  (Gemini views ‚Üí Meshy). Writes `Assets/Resources/Models/<id>.bytes` (`.bytes` REQUIRED ‚Äî Unity
  only ships known extensions via `Resources.Load`; `ModelLoader`+glTFast reads at runtime).
  Idempotent, never a hard gate ‚Äî a failed model degrades to primitive.
  `node gen-models.mjs <manifest> [-o projDir] [-i idsubstr] [-F] [-d]`
- **`alpha_key.py`** ‚Äî REQUIRED post-pass. nano-banana paints a fake grey/white checkerboard
  instead of real alpha; this flood-fills the border background to true transparency (stops at
  the dark subject outline). Skip it ‚Üí grey boxes in-game.

## Design-doc skills

Two distinct skills, do NOT confuse:

- **`game-design`** ‚Äî pure knowledge/reference (`allowed-tools: Read/Glob/Grep`, no output). Core
  loop, GDD structure, balancing, player psychology. Consult, not a generator.
- **`game-design-document`** ‚Äî GDD *generator*. Concept ‚Üí 40‚Äì80pg `.docx`/`.pdf` + `.pptx` deck +
  one-pager, via Python (`python-docx`, `fpdf2`, `python-pptx`). Mandatory interactive interview
  ‚Üí stays a skill. All 4 generators + a shared `_config.py` live in `scripts/`; they consume one
  `game_config.json` (schema + `game_config.example.json` in the skill). Interview ‚Üí write the
  config ‚Üí run the scripts from the skill dir (so `_config.py` imports resolve).

## unity-poc pipeline

Worker-less: game brief ‚Üí design docs ‚Üí code-driven Unity project ‚Üí WebGL build ‚Üí public Vercel URL. Everything spawns from C# at runtime (one empty boot scene, no hand-authored scenes or prefabs). Ships two bundled fighter frameworks.

### Pipeline phases (skills load in the main loop; agents spawned via Task tool)

| phase | kind | name | steps | output |
|-------|------|------|-------|--------|
| Spec | skill | `unity-poc-spec` | 1‚Äì3 | `PRD.md` ‚Üí `TDD.md` ‚Üí `ASSETS.md` / `sprites.manifest.json` (roster) / `assets.manifest.json` / `models.manifest.json` |
| Scaffold | agent | `unity-scaffold` | 4‚Äì5 | Unity project created headlessly; `templates/fighter2d/` or `templates/arena3d/` copied |
| Assets | agent | `unity-assets` | 6 | roster ‚Üí `Resources/Art/` via `gen-sprites.mjs` (real alpha, no keyer); scenery/UI/FX via `gen-assets.mjs` + `alpha_key.py`; GLBs ‚Üí `Resources/Models/` (3D). Never a hard gate ‚Äî missing art falls back to flat color / primitive capsule |
| Gameplay | skill | `unity-poc-gameplay` | 7 | `Assets/Scripts/Game/` authored; `BuildRoster()` exposed |
| Build+Ship | agent | `unity-buildship` | 8‚Äì13 | Headless playtest gate ‚Üí WebGL build ‚Üí local puppeteer boot test ‚Üí `deploy-vercel.sh` ‚Üí `HANDOFF.md` |

Spec/gameplay stay skills (interactive scope, code authoring). Scaffold/assets/buildship are
agents ‚Äî non-interactive, noisy logs isolated. The `unity-buildship` agent runs
`deploy-vercel.sh` directly (can't spawn the `portal-deploy` agent); `portal-deploy` stays for
standalone/3d-model publishes from the main loop.

### Template pick

| brief | template | namespace | assets |
|-------|----------|-----------|--------|
| 2D fighter / arcade | `templates/fighter2d/` | `Fighter` | roster sprites (`gen-sprites.mjs`) + scenery/UI (`gen-assets.mjs`) |
| 3D arena brawler | `templates/arena3d/` | `Fighter3D` | Meshy GLB models (`gen-models.mjs` ‚Üí glTFast) |
| platformer / cozy / other | ‚Äî | your own | write from scratch; reuse pipeline + scripts only |

Genres are **self-contained folders under `templates/`** (no shared C# core ‚Äî 2D/3D already
diverge in loader/camera/fallback). Add a new genre by copying one and honoring the two Editor
entry methods + `BuildRoster()` reflection contract in
`skills/unity-poc/references/genre-contract.md`.

### Key gotchas

- **IL2CPP stripping** ‚Äî "Could not produce class with ID N" at boot. Build uses `ManagedStrippingLevel.Minimal` + `stripEngineCode = false`. Only the puppeteer browser test catches this.
- **nano-banana never returns real alpha** ‚Äî run `alpha_key.py` (REQUIRED for transparent sprites); skipping shows grey checkerboard boxes.
- **WebGL compression Disabled** ‚Äî intentional for plain static hosting.
- **Generated art must live under `Resources/Art/`** (2D) or `Resources/Models/<id>.bytes` (3D) ‚Äî only `Resources/` ships with `Resources.Load`.
- **glTFast is optional** ‚Äî real 3D models need BOTH `com.unity.cloud.gltfast` package AND `Assets/csc.rsp` with `-define:HAS_GLTFAST`. Add the define only when the package is present.
- **`*.sh` scripts default to `Fighter.*`** ‚Äî 3D builds must set `BUILD_METHOD`/`PLAYTEST_METHOD=Fighter3D.*`.
- Two hard gates: **playtest** (aborts build on fail) and **local puppeteer browser test** (aborts deploy on fail). Both fail back to `unity-poc-gameplay`.
- One soft gate: **visual review** ‚Äî buildship runs `gameplay-shots.mjs` (real play inputs ‚Üí frames) and the orchestrator vision-reviews them before calling the ship done; boot-passing builds have shipped checkerboarded FX and invisible UI. Glow FX/UI: gen on black + `fx_luma_key.py`, never `alpha_key.py` (its flood can't cross the fake checkerboard).

### Output location

`unity/output/<project-name>/` ‚Äî gitignored, regenerated by pipeline. Each project folder contains `PRD.md`, `TDD.md`, `ASSETS.md`, `HANDOFF.md`, build logs, and the asset manifest.
